这份总体大纲剥离了重型游戏引擎的依赖，采用前后端分离的现代化桌面应用架构。整体设计强调模块化，方便你独立进行接口调试与功能扩展。

以下是为你定制的 AI 桌宠全栈开发大纲：

### 项目概览：模块化智能桌面伴侣
**核心目标：** 开发一个具备深度定制人设、高保真克隆音色，并能通过函数调用（Function Calling）执行日常任务的桌面级智能体。
**架构核心：** Electron 负责桌面级展现，Vue3 负责 UI 交互，Python 承担智能体逻辑调度。

---

### 一、 系统架构设计

系统将分为三个独立解耦的层级进行构建：

* **表现层 (前端 UI & 渲染)：** 负责呈现透明无边框窗口、Live2D 角色渲染、消息气泡弹窗以及 To-do/邮件的 UI 面板。
* **控制层 (智能体核心驱动)：** 作为系统的大脑中枢，负责解析语音/文本输入、维护对话历史与人设状态、调度大模型并决策何时调用外部工具。
* **基建层 (AI 模型服务)：** 提供底层 AI 能力支撑，包含大语言模型推理接口、语音识别（STT）和语音合成（TTS）服务。

---

### 二、 技术栈选型建议

* **桌面端外壳：** Electron 
* **前端框架：** Vue3 (配合 TypeScript 构建交互面板与状态管理)
* **角色渲染组件：** PixiJS + Live2D Cubism SDK
* **后端调度器：** Python + FastAPI (构建轻量级 RESTful API)
* **大模型框架：** LangChain 或通过原生 MCP 协议对接
* **语音合成端：** GPT-SoVITS-V2 (独立端口提供 HTTP 服务)

---

### 三、 核心模块功能拆解

#### 1. 人设与记忆引擎 (Profile & Memory)
* **角色卡解析器：** 读取符合 V2 标准的 JSON 文件，提取角色的 `Personality`、`Backstory` 等字段。
* **Prompt 构造器：** 动态拼接系统提示词。每次对话前，将角色背景与当前时间、系统环境信息合并送入 LLM。
* **记忆挂载：** 使用本地向量数据库（如 ChromaDB）或简单的 JSON 日志文件，存储用户的长期偏好与历史对话记录。

#### 2. 外部任务处理系统 (Tool Use / Agentic Workflow)
* **意图拦截：** 在 LLM 返回文本前，通过系统级 Prompt 引导模型判断是否需要调用外部工具。
* **To-do List 管理器：** 封装增加、查询、删除待办事项的 Python 函数。数据可存储于本地 SQLite。前端 Vue3 组件通过轮询或 WebSocket 实时更新并渲染任务列表。
* **邮件监控器：** 封装基于 IMAP 协议的邮件读取脚本，实现定时拉取未读邮件列表。当捕获到新邮件时，主动向前端推送系统通知，触发桌宠的“主动说话”行为。

#### 3. 视听同步交互链路 (Audio-Visual Sync)
* **全双工监听：** 接入 VAD（语音活动检测），精准识别用户发言的起止时间，配合本地 Whisper 模型实现语音转文字。
* **流式语音生成：** 接收 LLM 生成的文本流，逐句发送至本地部署的 GPT-SoVITS 服务进行音频合成。
* **动效映射：** 提取音频的音量大小计算口型开合度（Lip-sync），同步驱动 Live2D 模型的唇部参数。

---

### 四、 阶段性开发路线图 (Milestones)

**第一阶段：跑通核心对话链路 (MVP)**
1.  搭建前端 Vue3 + Electron 透明窗口，加载基础 Live2D 模型。
2.  编写 Python 脚本，打通“文本输入 -> LLM API -> 文本输出”的基础接口。

**第二阶段：注入灵魂与声音**
1.  基于语料训练 GPT-SoVITS 音色模型并暴露为本地 API。
2.  完善角色卡读取逻辑，实现文本流与音频流的打通。
3.  处理前端音频播放与模型口型的同步逻辑。

**第三阶段：赋予任务执行能力**
1.  在 Python 端编写邮件拉取与 To-do 管理的核心函数。
2.  调整 LLM 提示词，实现 Function Calling 逻辑，让桌宠能根据对话意图自主调用上述函数。
3.  在前端 Vue3 中开发对应的可视化展示面板。

**第四阶段：打磨与桌面级体验优化**
1.  实现鼠标点击穿透、拖拽移动以及系统托盘常驻。
2.  优化内存占用，增加“待机模式”降低休眠时的性能开销。

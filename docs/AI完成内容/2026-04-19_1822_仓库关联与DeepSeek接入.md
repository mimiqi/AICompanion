## 三、关键改动详情

### 3.1 conf.yaml LLM 配置

`character_config.agent_config.agent_settings.basic_memory_agent` 当前选用 `openai_compatible_llm` provider，因此直接修改 provider 配置即可生效，不动 agent 选择字段。

修改前：

```yaml
openai_compatible_llm:
  base_url: 'http://localhost:11434/v1'  # Ollama 默认
  llm_api_key: 'ollama-no-need'
  model: 'qwen2.5:latest'
  temperature: 1.0
```

修改后：

```yaml
openai_compatible_llm:
  base_url: 'https://api.deepseek.com/v1'
  llm_api_key: 'sk-************'           # 真实 key 已写入本地 conf.yaml
  model: 'deepseek-chat'                    # 也可改 'deepseek-reasoner' (R1)
  temperature: 0.7                           # DeepSeek 范围 0~1
```

`deepseek_llm` 块的 `llm_api_key` 也同步填入，便于将来 `llm_provider` 直接切到 `deepseek_llm`。

### 3.2 uv PATH 修复

uv 官方安装脚本 (`https://astral.sh/uv/install.ps1`) 已自动写入用户 PATH，但：
- **当前 PowerShell 窗口启动早于安装** → 进程内 PATH 不刷新 → `uv` 找不到
- 安装脚本被多次执行，`C:\Users\mimiqi\.local\bin` 在用户 PATH 里**重复 3 次**

解决方式：去重并保持靠前位置。临时生效用 `$env:Path = "C:\Users\mimiqi\.local\bin;$env:Path"`，永久生效靠注册表（已完成）。

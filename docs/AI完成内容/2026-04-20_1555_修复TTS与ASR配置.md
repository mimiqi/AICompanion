## 三、根因详细分析

### Bug 1:听到 edge_tts 中文女声而非利托里奥 SoVITS

**事故链**:
- 4-19 18:33 通链验证准备(Scope B)阶段:把 `tts_model` 从 `'gpt_sovits_tts'` 临时改成 `'edge_tts'`(因为当时 SoVITS 没训完、ref_audio 是空,启动会崩),并切到 `zh-CN-XiaoxiaoNeural` 中文女声
- 4-19 21:06~22:42 日语桌宠双轨方案 + e50 切换:改了 6 处 conf.yaml(persona/asr/translator/text_lang/prompt_lang/ref_audio/prompt_text),**但漏了把 tts_model 这一行切回去**
- 4-20 15:51 用户启动 OLV:OLV 看到 `tts_model: 'edge_tts'`,自然走 edge_tts 中文女声链路;翻译模块仍工作(腾讯把"我是利托里奥"翻成"こちらはLitorio!"),但日语文本被喂给中文女声模型,听感就是"中文女声念了几个怪音节"

**证据**:用户 15:51 终端日志显示翻译完美:
```
🏃 Text after translation: '''こちらはLitorio!'''...
🏃 Text after translation: '''地中海最強の戦艦。'''...
```
但全程**没有任何 `:9880` / `gpt_sovits` 调用日志**,说明 OLV 根本没走 SoVITS API。

### Bug 2:faster-whisper 不接受 `'auto'`

**事故链**:
- 4-19 21:06 日语桌宠双轨方案:基于错误假设,把 ASR `language` 从 `'zh'` 改成 `'auto'`(以为 faster-whisper 像某些云端 ASR 那样支持 auto 关键字)
- 4-20 15:51 用户对着麦克风说话:OLV 把 `'auto'` 字符串原样传给 faster-whisper -> 在它的语言代码字典里找不到 -> `ValueError`

**真相**:faster-whisper 想"自动检测"必须传 `language=None`(Python None,不是空字符串),且需要 OLV wrapper 支持。OLV 当前 wrapper(`faster_whisper_asr.py`)直接把 yaml 字符串传过去,所以**只能填具体 ISO 639-1 代码**(`zh`/`ja`/`en` 等)。

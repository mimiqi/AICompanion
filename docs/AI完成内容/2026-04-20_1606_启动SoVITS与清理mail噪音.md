## 四、为什么要拿掉 mail MCP

每次 OLV 启动 + 每次浏览器新建 WebSocket 会话时,OLV 都会尝试启动 mail MCP server,而 [mcp_servers/mail_server/server.py](../../mcp_servers/mail_server/server.py) 在 `__main__` 里就 `MailConfig.load()`,因为 [mail_config.json](../../mcp_servers/mail_server/mail_config.json) 不存在所以**进程启动后立刻 raise FileNotFoundError 自杀**。MCP 客户端看到子进程进程死了,就报 `Connection closed`,然后 OLV 接住异常打 5 段 traceback。

**单次启动会刷出 ≥ 15 段 traceback**(初始化 + load_cache + 每次新连接重试)。这些是无害噪音,但视觉上把真正的错误(比如本次的 WinError 10061)埋得很深 —— 你需要往下翻几屏才能看到关键信息。

拿掉 mail 后:
- OLV 启动日志干净一大片(短一半)
- 真正的 ERROR(如 SoVITS 调不通)能一眼看到
- 桌宠依然能用 `time`(查时间) + `todo`(查/加待办) MCP 工具
- 等以后想用邮件功能,填好 [mail_config.json](../../mcp_servers/mail_server/mail_config.json) IMAP 凭据 + 把 `"mail"` 加回去即可

### Phase F:翻译质量调优(可选,听 5~10 条对话后决定)

当前方案的质量上限是 "**利托里奥的音色,但说话像 NHK 主播**" — 音色像、字幕中文 OK,只在 "日语自然度" 这一栏吃了亏。腾讯翻译 zh→ja 在动漫角色对话上偶有 3 类典型偏差:

| 偏差 | 例子 | 后果 |
|---|---|---|
| **翻译腔** | "指挥官,你今天怎么样?" -> 「指揮官、今日はどうでしたか?」(过于书面;真人会说「司令官、今日どうだった?」) | 利托里奥变得"过于客气",失去傲娇语气 |
| **专有名词漂移** | "指挥官" -> 「コマンダー」(片假名);游戏里其实用「指揮官」 | 沉浸感下降 ~10% |
| **语气词丢失** | "哎呀指挥官,我还想再睡会儿呢" -> 没有 「あら、まだ寝足りないなぁ」 那种自然感叹词 | 利托里奥少了几分"鲜活感" |

**评估方法**:训完模型 + 启用 OLV 后,跑 5~10 条对话试听。然后按下面三档之一升级:

#### 升级 1:Persona 微调(免费,5 分钟,效果 -30% 翻译腔)

如果体感"有点翻译腔但能接受",在 [Open-LLM-VTuber/conf.yaml](../../Open-LLM-VTuber/conf.yaml) `persona_prompt` 末尾追加:

```yaml
persona_prompt: |
  ...(原内容保留)...
  为了让翻译后的日语更自然,请遵守:
  - 多用感叹词("哎"、"哦"、"嗯"、"嘿")
  - 单句尽量 ≤ 12 个汉字
  - 称呼固定用"指挥官"(避免"先生"、"老板"、"亲")
  - 避免"是吧"、"对吧"等模糊语气词,改用直球句末("呢" / "啊" / "哦")
  - 避免成语、网络用语、英文缩写
```

预期:翻译腔下降 ~30%,无任何额外延迟/成本。

#### 升级 2:DeepSeek 当翻译引擎(中等,~50 行 Python,效果 -70% 翻译腔)

如果升级 1 后仍觉得"明显机翻味,不像利托里奥",新建 [Open-LLM-VTuber/src/open_llm_vtuber/translate/llm_translator.py](../../Open-LLM-VTuber/src/open_llm_vtuber/translate/llm_translator.py),实现 `TranslateInterface`:

```python
class LLMTranslator(TranslateInterface):
    def __init__(self, base_url, api_key, model='deepseek-chat'):
        from openai import OpenAI
        self.client = OpenAI(base_url=base_url, api_key=api_key)
        self.model = model
        self.sys_prompt = (
            "把下面这句中文翻译成动漫风格的口语日语。要求:\n"
            "- 保留原文的傲娇/可爱/调侃语气\n"
            "- 用感叹词,避免书面语\n"
            "- 称呼固定用「指揮官」\n"
            "- 直接输出译文,不要解释、不要引号"
        )

    def translate(self, text: str) -> str:
        rsp = self.client.chat.completions.create(
            model=self.model,
            messages=[
                {"role": "system", "content": self.sys_prompt},
                {"role": "user", "content": text},
            ],
            max_tokens=200, temperature=0.6,
        )
        return rsp.choices[0].message.content.strip()
```

并在 [src/open_llm_vtuber/service_context.py](../../Open-LLM-VTuber/src/open_llm_vtuber/service_context.py) 的 `init_translate` 里注册 `'llm'` provider 分支。conf.yaml:

```yaml
translator_config:
  translate_audio: True
  translate_provider: 'llm'
  llm:
    base_url: 'https://api.deepseek.com/v1'
    llm_api_key: 'sk-...'   # 复用现有 DeepSeek key
    model: 'deepseek-chat'
```

预期:翻译腔下降 ~70%,延迟 +0.5~1.5s,API 成本 +30%。

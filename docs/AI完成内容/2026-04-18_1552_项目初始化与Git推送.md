### M2-2 - Character Card V2 解析器

新建 `Open-LLM-VTuber/src/open_llm_vtuber/agent/persona/`：

- `__init__.py`：导出 `CharacterCardV2`、`CharacterCardV2Loader`
- `character_card_v2.py`：完整解析 SillyTavern V2 规范（兼容 V1 平铺布局），支持 `{{user}}/{{char}}` 占位符替换、`<START>` 分隔的 few-shot 多轮示例对话；提供 `build_system_prompt()`、`build_few_shot_messages()`、`build_initial_greeting()` 三个渲染入口

新建 `characters_v2/default.json`：示例角色卡 "Aki"，作为首次运行验证用例。
新建 `characters_v2/README.md`：说明 V2 字段含义和从 SillyTavern 导入流程。

### M2-3 - CompanionAgent

新建 `Open-LLM-VTuber/src/open_llm_vtuber/agent/agents/companion_agent.py`：

- 继承 `BasicMemoryAgent`，复用全部 LLM/MCP/工具调用流水线
- 在 `__init__` 中加载 Character Card V2，渲染为 system prompt 注入父类
- 重写 `_to_messages`：自动注入 few-shot 示例、根据当前 query 从长期记忆召回 top-k 拼到 messages 末尾
- 重写 `_add_message`：除了短期 buffer 外，同时写入 long-term store
- `set_memory_from_history` 时标记 few-shot 已注入，避免与历史上下文冲突
- 长期记忆为可选依赖（`Optional[MemoryStore]`），无 ChromaDB 时降级为 BasicMemoryAgent 行为

修改 `agent_factory.py`：

- import `CompanionAgent`
- 新增 `conversation_agent_choice == "companion_agent"` 分支，处理 ChromaDB 实例化（缺失时优雅降级）
- 复用现有 LLM provider / tool_manager / tool_executor / mcp_prompt_string 注入

修改 `config_manager/agent.py`：

- 新增 `CompanionAgentConfig`（含 `character_card_path`、`user_name`、`chroma_persist_dir`、`memory_top_k`、`short_term_window` 等字段）
- `AgentSettings` 注册 `companion_agent: Optional[CompanionAgentConfig]`
- `AgentConfig.conversation_agent_choice` 的 `Literal` 加入 `"companion_agent"`

### M3 - ChromaDB 长期记忆

新建 `Open-LLM-VTuber/src/open_llm_vtuber/agent/memory/`：

- `base.py`：定义 `MemoryRecord` dataclass + 抽象 `MemoryStore`（add / query / reset）
- `chroma_store.py`：`ChromaMemoryStore` 使用 `chromadb.PersistentClient`，cosine 相似度，可选 SentenceTransformer 嵌入；优雅处理 ImportError（chromadb 未装时由 agent_factory 接住并降级）
- `__init__.py`：lazy import 设计，确保 chromadb 不是硬依赖

### M4-1 - To-do MCP Server

新建 `mcp_servers/todo_server/`：

- `__init__.py`（空）
- `db.py`：线程安全的 SQLite 包装，schema 含 `id/title/notes/status/due_at/created_at/updated_at/completed_at` + 两个索引
- `server.py`：`mcp.server.fastmcp.FastMCP` 实例 `todo`，暴露 6 个工具
  - `add_todo(title, notes?, due_at?)`：支持 ISO 8601 时间字符串
  - `list_todos(status='pending'|'completed'|'all', limit=20)`
  - `complete_todo(todo_id)`
  - `update_todo(todo_id, title?, notes?, due_at?, status?)`
  - `delete_todo(todo_id)`
  - `stats()`：返回汇总 + 当前已逾期项（用于桌宠主动催办）
- `README.md`：使用文档 + `uvx mcp dev` 调试方式

### M4-2 - Mail MCP Server + 主动播报守护

新建 `mcp_servers/mail_server/`：

- `__init__.py`（空）
- `config.py`：`MailConfig` 加载器 + 状态文件读写助手
- `imap_client.py`：纯 stdlib (`imaplib` + `email`)，自动解码 multipart 正文与 RFC 2047 Header
- `server.py`：4 个 MCP 工具
  - `fetch_recent_emails(unread_only=True, limit=10, include_body=False)`
  - `get_email_detail(uid)`
  - `get_unread_count()`
  - `get_mail_account_info()`
- `daemon.py`：独立守护进程，按配置间隔轮询 IMAP，发现新邮件时通过 `websocket-client` 连接 OLV 的 `/client-ws` 并发送 `{"type": "ai-speak-signal"}` 触发桌宠主动开口；指数退避重连，UID 状态持久化避免重启重复触发，支持发件人白名单
- `mail_config.example.json`：完整示例配置
- `README.md`：详细安全说明（强烈建议使用 App Password）

## 六、用户后续需要手动完成的事

1. 安装 `uv`，进入 `Open-LLM-VTuber` 跑 `uv sync`，再 `uv pip install chromadb`
2. `cp conf.example.yaml conf.yaml` 并填入 LLM API key
3. `cp mcp_servers.example.json mcp_servers.json`
4. 训练 GPT-SoVITS-V2 模型并配置 `gpt_sovits_tts.ref_audio_path` / `prompt_text`
5. 启用邮件功能：`cd mcp_servers/mail_server && cp mail_config.example.json mail_config.json` 并填入 IMAP 凭据
6. 切换到 CompanionAgent：编辑 `conf.yaml` 把 `conversation_agent_choice` 改为 `'companion_agent'`
7. 前端面板：`cd frontend_overlay/upstream && npm install && npm run build:web`，按 `frontend_overlay/README.md` 把构建产物覆盖到 `Open-LLM-VTuber/frontend/`

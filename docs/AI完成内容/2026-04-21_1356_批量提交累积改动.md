## 四、坑与教训

### 坑 1:gitignore 例外行不能有行内注释

第一次写的 `Open-LLM-VTuber/.gitignore` 加例外:
```gitignore
!/backgrounds/ceiling-window-room-night.jpeg   # 占位图,前端 bundle...
```

`git check-ignore -v` 显示**例外没生效**,因为 git 把整行(包括 `#` 之后的部分)当作 pattern。修复:把注释拆到独立的前一行:

```gitignore
# 占位图: 前端 bundle 硬编码默认请求此文件名,缺则 404,故纳入版本控制
!/backgrounds/ceiling-window-room-night.jpeg
```

`git status` 立刻列出该文件 = 例外生效 ✅

### 坑 2:`git check-ignore` 在 negation 规则上的反直觉

执行 `git check-ignore -v <bg jpeg>` 即使例外生效后,**仍然返回 exit code 0** 并显示 `!/backgrounds/...` 的匹配。这容易误以为"还是被 ignore"。

**真理**:看 `git status --short` 是否列出该文件。`check-ignore` 只是告诉你哪条规则匹配,negation 也算匹配。

### 坑 3:PowerShell 不支持 bash heredoc

最初尝试 `git commit -m "$(cat <<'EOF' ... EOF)"` 直接报 ParserError。改用 `git commit -F <file>` 把 message 写入临时文件 `agent-tools/commit_msg.txt`,完美解决多行 + 中文 + 特殊字符的所有麻烦。

### 3.2 背景图占位生成

PowerShell 命令:

```powershell
Add-Type -AssemblyName System.Drawing
$bmp = New-Object System.Drawing.Bitmap 1920, 1080
$g = [System.Drawing.Graphics]::FromImage($bmp)
$g.Clear([System.Drawing.Color]::FromArgb(20, 22, 32))   # 暗夜蓝灰
$bmp.Save(
    'D:\CodeField\CODE_PYTHON\AICompanion\Open-LLM-VTuber\backgrounds\ceiling-window-room-night.jpeg',
    [System.Drawing.Imaging.ImageFormat]::Jpeg
)
$g.Dispose(); $bmp.Dispose()
```

生成后:
- 文件大小:33,268 bytes
- 文件头:`FF D8 FF E0`(JFIF/JPEG 标准头)✅
- 颜色:RGB(20, 22, 32) 暗夜蓝灰,不抢桌宠风头

**为什么不改前端默认背景文件名而是补占位**:[bgurl-context.tsx](../../Open-LLM-VTuber/frontend_overlay/upstream/src/renderer/src/context/bgurl-context.tsx) 第 45 行 `DEFAULT_BACKGROUND` 已编进 [Open-LLM-VTuber/frontend/assets/main-nu7uwxNJ.js](../../Open-LLM-VTuber/frontend/assets/main-nu7uwxNJ.js) bundle,改名要重新构建前端(本计划范围外)。补占位是最低成本方案。

## 五、风险与回退

### 风险 1:某个 emo 切到的表情明显不合适

mao_pro 的 8 个 expression 文件名是通用编号(`exp_01.exp3.json` ~ `exp_08.exp3.json`),**没有"happy"/"sad" 语义标签**。也就是:
- 我们假设 "joy → 1 → exp_02 是个微笑"可能不对
- 实际可能 exp_02 是闭眼或抬眉
- 试错最快:LLM 触发某个 emo 时桌宠如果出现"愤怒切到笑脸"这种明显不对的情况,**告诉我哪个 emo + 实际表情感觉**,我对应换一个 0-7 之间的数字即可

### 回退方法

```bash
git diff HEAD -- Open-LLM-VTuber/model_dict.json
# 看到改动后想还原:
git checkout Open-LLM-VTuber/model_dict.json
```

(或手工把 emomap 改回之前 4 槽方案)

### 风险 2:背景过于素雅

暗夜蓝灰是为了不抢桌宠风头,但你可能想要"有意境的"背景。后续 2 个换法:

a) 你在 [Open-LLM-VTuber/backgrounds/](../../Open-LLM-VTuber/backgrounds/) 丢任意 jpg/png 进去,然后浏览器右上角设置 → 背景里选(localStorage 会记住)

b) 直接覆盖 `ceiling-window-room-night.jpeg` 为别的图(保持文件名,bundle 默认请求依然 200)
